# User Journey Mapping

## Journey A — Membaca Artikel (Pembaca Panjang)

1. Masuk landing/home
2. Memilih kategori/topik
3. Membuka artikel
4. Membaca hingga bawah
5. Simpan/bagikan

**Validasi desain**
- Home menonjolkan jalur “baca” (artikel terbaru/kategori).
- Header artikel: judul + meta stamp terbaca cepat.
- Konten: lebar kolom nyaman, line-height 1.65, spacing antar paragraf jelas.
- CTA share/simpan tidak memotong ritme membaca.

## Journey B — Scan Cepat (Pencari Info Cepat)

1. Masuk home
2. Scan judul + excerpt (F-pattern)
3. Filter kategori
4. Buka 2–3 artikel kandidat
5. Keputusan: baca penuh / kembali

**Validasi desain**
- Kartu artikel punya struktur konsisten: badge → judul → ringkas → meta.
- Kontras dan whitespace membuat scanning cepat.
- Tab kategori dan state aktif jelas.

## Journey C — Aksi (Pendukung/Relawan)

1. Masuk home
2. Menangkap value proposition (headline + subcopy)
3. Melihat impact/stat
4. Klik CTA (gabung/donasi/kontak)
5. Selesai aksi

**Validasi desain**
- CTA primary muncul sekali sebagai pilihan utama.
- Elemen trust/impact menggunakan angka besar + label ringkas.
- Warna accent tidak mengacaukan fokus CTA.
